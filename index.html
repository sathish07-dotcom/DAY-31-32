ASYNC`& AWAIT

In JavaScript, `async` and `await` are used for handling asynchronous operations in a more readable and cleaner way, compared to using callbacks or promises directly. Here's how they work:

async

  • The `async` keyword is used to define a function that returns a `Promise`.
  • Any function that is marked as `async` will implicitly return a `Promise`. If the function returns a value, that value is automatically wrapped in a resolved `Promise`. If it throws an error, the `Promise` is rejected.

    • async function example() {
    return "Hello, World!";
    }
    example().then(console.log); // Outputs: Hello, World!

    await

  • The `await` keyword can only be used inside an `async` function.
  • It makes JavaScript wait until the `Promise` is settled (either resolved or rejected) before moving to the next line of code.
  • It simplifies the code, making it look synchronous, even though it's running asynchronously behind the scenes.

    • async function fetchData() {
    let response = await fetch('https://jsonplaceholder.typicode.com/posts');
    let data = await response.json();
    return data;
    }
    fetchData().then(data => console.log(data)); // Outputs data from the API

    Putting it together

    async function getUserData(userId) {
    try {
    let response = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`);
    let user = await response.json();
    console.log(user);
    } catch (error) {
    console.error('Error:', error);
    }
    }
    getUserData(1); // Fetches and logs user data from the API

    Key Points:

  • `async` functions always return a `Promise`.
  • `await` is used to wait for a `Promise` to resolve or reject before continuing.
  • Using `async/await` makes the code easier to read and work with, especially when you need to perform multiple asynchronous operations in a sequence.
